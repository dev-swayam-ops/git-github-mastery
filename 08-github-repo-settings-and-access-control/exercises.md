# 08-github-repo-settings-and-access-control: Exercises

## Exercise 1: Configure Basic Repository Settings
**Difficulty:** Easy

Explore and update your repository's basic settings.

**Instructions:**
1. Navigate to Settings tab of any repository
2. Update repository description
3. Add repository URL
4. Enable/disable Wikis
5. Enable/disable Projects
6. Save changes

**Expected Outcome:** Repository settings updated successfully

---

## Exercise 2: Manage Repository Visibility
**Difficulty:** Easy

Change repository visibility settings.

**Instructions:**
1. Go to repository settings
2. Find "Danger Zone" section
3. Check current visibility (Public/Private)
4. Note: Don't change if it affects your work
5. Understand implications of each setting

**Expected Outcome:** Understand visibility implications

---

## Exercise 3: Add Collaborators to Repository
**Difficulty:** Easy

Grant access to team members.

**Instructions:**
1. Go to Settings > Collaborators and teams
2. Click "Add people"
3. Search for GitHub username
4. Select role (Maintain, Write, Triage, Read)
5. Understand each permission level

**Expected Outcome:** Know how to add collaborators

---

## Exercise 4: Understand Repository Roles
**Difficulty:** Medium

Learn different access levels and permissions.

**Instructions:**
1. Review each role in documentation:
   - Read (Pull only)
   - Triage (Pull + Issues)
   - Write (Read + Push + Tests)
   - Maintain (Write + Admin)
   - Admin (Full access)
2. Create a table showing which role for which task
3. Identify appropriate role for different team members

**Expected Outcome:** Understand role-based access control

---

## Exercise 5: Configure Default Branch
**Difficulty:** Medium

Set which branch is used as default.

**Instructions:**
1. Go to Settings > Branches
2. Check current default branch
3. Switch default to another branch
4. Understand impact on PRs and clones
5. Switch back to original

**Expected Outcome:** Know how default branch affects workflows

---

## Exercise 6: Set Up Branch Protection Rules (Basic)
**Difficulty:** Medium

Configure protection for important branches.

**Instructions:**
1. Go to Settings > Branches
2. Click "Add rule"
3. Specify branch name pattern (e.g., main)
4. Enable "Require pull request reviews"
5. Enable "Require status checks"
6. Save rule

**Expected Outcome:** Branch protection rule created

---

## Exercise 7: Manage Teams and Permissions
**Difficulty:** Medium

Work with team-level access control.

**Instructions:**
1. Go to Settings > Teams
2. Add existing team to repository
3. Assign team role
4. Understand team vs individual permissions
5. Compare with individual collaborators

**Expected Outcome:** Know differences between team and individual access

---

## Exercise 8: Configure Webhook Settings
**Difficulty:** Medium

Set up webhooks for external integrations.

**Instructions:**
1. Go to Settings > Webhooks
2. Examine webhook configuration options
3. Understand trigger events
4. Know what payload is sent
5. Don't create live webhook without endpoint

**Expected Outcome:** Understand webhook purposes

---

## Exercise 9: Review Deploy Keys
**Difficulty:** Medium

Understand deploy keys for CI/CD.

**Instructions:**
1. Go to Settings > Deploy keys
2. Understand difference between deploy keys and personal tokens
3. Know use cases for each
4. Understand read-only vs read-write options
5. See how they're used in CI/CD

**Expected Outcome:** Know when to use deploy keys

---

## Exercise 10: Configure Repository Automation
**Difficulty:** Medium

Set up automatic management features.

**Instructions:**
1. Go to Settings and explore:
   - Auto-merge options
   - Squash/Rebase defaults
   - Branch deletion on merge
   - Require status checks
2. Understand each setting
3. Configure for your workflow

**Expected Outcome:** Repository auto-configured for workflow
