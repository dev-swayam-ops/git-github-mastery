# 12-collaboration-labs-and-scenarios: Exercises

## Exercise 1: Simulated Team Collaboration
**Difficulty:** Medium

Practice working in a team on a single project.

**Instructions:**
1. Create test project with 2-3 team members
2. Each person creates feature branch
3. Make changes independently
4. Submit PRs simultaneously
5. Review each other's code
6. Merge without conflicts

**Expected Outcome:** Successfully merged all features

---

## Exercise 2: Conflict Resolution Scenario
**Difficulty:** Medium

Handle merge conflicts in collaborative environment.

**Instructions:**
1. Two people edit same file
2. Create conflicting changes
3. Attempt merge
4. Resolve conflicts together
5. Verify resolution
6. Complete merge

**Expected Outcome:** Conflicts resolved, merged successfully

---

## Exercise 3: Code Review Process
**Difficulty:** Medium

Practice complete code review workflow.

**Instructions:**
1. Person A creates feature PR
2. Person B reviews, requests changes
3. Person A makes improvements
4. Person B approves
5. Person A merges
6. Document review feedback

**Expected Outcome:** Complete review cycle documented

---

## Exercise 4: CI/CD Pipeline in Team
**Difficulty:** Medium

Set up automated checks for team collaboration.

**Instructions:**
1. Create GitHub Actions workflow
2. Require CI passing for merge
3. Each PR runs tests automatically
4. Failing tests block merge
5. Team sees status checks
6. Only approved + passing PRs merge

**Expected Outcome:** Automated quality gate working

---

## Exercise 5: Branching Strategy Implementation
**Difficulty:** Medium

Implement Git Flow strategy with team.

**Instructions:**
1. Designate main/develop branches
2. Create feature branches from develop
3. Create release branch
4. Create hotfix from main
5. Merge back appropriately
6. Tag releases

**Expected Outcome:** Git Flow strategy working

---

## Exercise 9: Emergency Hotfix Process
**Difficulty:** Medium

Practice urgent production fixes.
